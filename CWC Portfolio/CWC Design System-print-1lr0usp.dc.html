18

Spacing, alignment & padding rules

Baseline grid
  • All spacing (margin, padding, gap) is a multiple of 8px. 4px is allowed only as a micro-adjustment inside a component (icon-to-label gaps, badge padding), never between components.
  • Section padding is fixed at 56px top and 64px sides, for every section, no exceptions.
  • A number badge and its heading share one row set to align-items: center, not baseline — baseline alignment misaligns a padded badge against cap-height text.
  • Vertical rhythm between a section's parts: 8px label→heading, 24px heading→intro copy, 32–40px intro→content.
Alignment
  • Left edges lock to the 12-col grid — never eyeball an offset.
  • Group a label with its value on one baseline (e.g. "Year — 2026"); don't split them across lines.
  • Icons sit in a fixed-size box (24px or 1em) and center visually with their adjacent text, not just their bounding box.
  • Dividers (dashed or solid) sit exactly between two 8px-multiple paddings — never floating mid-gap.
Padding — components
  • Buttons: 14px vertical / 28px horizontal, sized by padding, never a fixed width — text always gets room to breathe. Minimum 48px tap height.
  • Cards: 12–14px inner padding at all sizes; label sits flush to the bottom-left.
  • Chips / command badges: 4px vertical / 8–9px horizontal.
  • Containers (palette, detail panel): 24–32px padding, scaled down (~24px) only for a compact/overlay context.
